
### 1. "hello"、&"hello"、^"hello"的区别

在 Godot 4（含以后的 3.x LTS 版本）里的 GDScript 中，  
"hello"、&"hello"、^"hello" 这三种写法代表三种不同的类型：

1. "hello" → String
2. &"hello" → StringName
3. ^"hello" → NodePath

`const indoorZoneName := &"IndoorZone"`

就是把常量 indoorZoneName 声明为一个 StringName，而不是普通的 String。

为什么要用 & 生成 StringName？

1. 性能  
    • String 每次比较都会按字符逐个比较。  
    • StringName 在创建时就被哈希+驻留（intern），以后比较时只比对哈希，几乎是常数时间，速度快得多。  
    • 如果你在信号名、动画名、输入动作名、字典键等位置频繁比较相同的字符串，改用 StringName 会明显减少 GC 压力和 CPU 消耗。
    
2. 内存复用  
    • 相同内容的 StringName 只会在内存里存一份，所有引用指向同一个内部实例。  
    • 普通 String 则各自占用独立内存。
    
3. 语法简洁  
    • 以前必须写 `StringName("IndoorZone")`；4.0 开始直接写 &"IndoorZone"，既直观又是编译期常量。
